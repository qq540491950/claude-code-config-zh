# 设计协作轻量模式

用于在不引入重流程的前提下，提高 UI 一致性与可交付质量。

## 何时激活

- 新增或重构页面/组件
- 交互流程变化（表单、按钮、弹窗、状态反馈）
- 设计稿落地或自驱设计实现

## 核心原则

1. **一致性优先**：颜色、间距、字号、圆角、阴影尽量复用同一套 token。
2. **状态完整**：空态、加载态、错误态、禁用态必须明确。
3. **可访问性**：可聚焦、可键盘操作、语义标签和可读提示齐全。
4. **反馈及时**：用户操作后有明确反馈（成功/失败/进行中）。

## 交付检查单

- [ ] 关键组件视觉规范一致（间距/字号/色彩）
- [ ] 关键交互具备加载与错误反馈
- [ ] 表单与按钮支持键盘与可读提示
- [ ] 核心流程在窄屏与宽屏均可用
- [ ] 变更说明包含“设计影响范围”
