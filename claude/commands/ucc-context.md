# Context 命令

切换工作模式上下文，调整 Claude 的行为风格。

> 为了团队与个人更易理解，也可直接使用三个显式命令：
> - `/ucc-context-dev`
> - `/ucc-context-review`
> - `/ucc-context-research`

## 用法

`/ucc-context <模式>`

## 可用模式

| 模式 | 说明 | 适用场景 |
|------|------|---------|
| `dev` | 开发模式 | 活跃开发、编码、构建功能 |
| `review` | 审查模式 | PR 审查、代码分析、质量检查 |
| `research` | 研究模式 | 探索、调查、学习新技术 |

## 各模式行为特点

### 开发模式 (dev)

- 先写代码，后解释
- 优先可工作的方案，而非完美方案
- 修改后运行测试
- 保持提交原子化

**优先级：** 让它跑起来 → 让它正确 → 让它整洁

### 审查模式 (review)

- 评论前完整阅读代码
- 按严重性排序问题（严重 > 高 > 中 > 低）
- 提出修复建议，不只是指出问题
- 检查安全漏洞

**审查清单：** 逻辑错误、边界情况、错误处理、安全问题、性能、可读性、测试覆盖率

### 研究模式 (research)

- 广泛阅读后再下结论
- 提出澄清性问题
- 边研究边记录发现
- 理解清晰前不写代码

**研究流程：** 理解问题 → 探索相关代码/文档 → 形成假设 → 用证据验证 → 总结发现

## 使用示例

```bash
# 切换到开发模式
/ucc-context dev

# 切换到审查模式
/ucc-context review

# 切换到研究模式
/ucc-context research
```

## 参数

$ARGUMENTS: dev | review | research

## 输出

切换模式后，Claude 会：
1. 确认当前模式
2. 说明该模式下的行为准则
3. 按新模式风格响应当前任务
