# /ucc-learn - 提取可复用模式

分析当前会话并提取值得保存为技能的模式。

## 触发

在会话中解决非平凡问题时随时运行 `/ucc-learn`。

## 提取内容

寻找：

1. **错误解决模式**
   - 发生了什么错误？
   - 根本原因是什么？
   - 如何修复的？
   - 这对类似错误可复用吗？

2. **调试技术**
   - 非明显的调试步骤
   - 有效的工具组合
   - 诊断模式

3. **变通方案**
   - 库的怪异行为
   - API 限制
   - 版本特定的修复

4. **项目特定模式**
   - 发现的代码库约定
   - 做出的架构决策
   - 集成模式

## 输出格式

在 `~/.claude/skills/learned/[pattern-name].md` 创建技能文件：

```markdown
# [描述性模式名称]

**提取时间:** [日期]
**上下文:** [适用情况的简要描述]

## 问题
[此模式解决什么问题 - 要具体]

## 解决方案
[模式/技术/变通方案]

## 示例
[代码示例（如适用）]

## 使用时机
[触发条件 - 什么应该激活此技能]
```

## 流程

1. 回顾会话以提取可提取的模式
2. 识别最有价值/可复用的见解
3. 起草技能文件
4. 保存前请用户确认
5. 保存到 `~/.claude/skills/learned/`

## 注意事项

- 不提取琐碎修复（拼写错误、简单语法错误）
- 不提取一次性问题（特定 API 中断等）
- 专注于能节省未来会话时间的模式
- 保持技能专注 - 每个技能一个模式
