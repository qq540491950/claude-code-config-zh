# claude-code-config-zh 改进建议

> 目标：在保持“精简可用”的前提下，提升一致性、跨 CLI 兼容性与自动化能力。

## 一、当前状态（已确认）

- 当前能力主链路完整：`rules + agents + commands + skills`
- 数量概览：
  - `agents/`：6
  - `commands/`：6
  - `rules/`：15
  - `skills/`：4
- 对 Go / Python / Vue 项目可直接使用。

## 二、主要差距（相对上游 everything-claude-code）

1. 自动化能力较弱：缺少 `hooks/`、`scripts/`、`tests/`。
2. 平台化分发能力缺失：缺少 `.claude-plugin/`、`.opencode/`、`.cursor/` 适配层。
3. 扩展面较窄：语言/场景覆盖偏少。
4. 文档与能力可能存在不一致风险（用户按文档预期找不到对应能力）。

## 三、改进建议（按优先级）

### P0：一致性修复（先做）

1. 补齐“文档声明但未实现”的能力：
   - 增加 `build-error-resolver` 代理
   - 增加 `/build-fix` 命令
2. 做一次“文档-目录-命令”对齐检查，确保 README 与实际能力一致。

**价值**：降低用户预期落差，减少“按文档操作失败”的问题。

### P1：语言覆盖补齐（第二阶段）

1. 增加独立 TypeScript 规则目录（非 Vue 场景）：
   - 建议新增：`rules/typescript/`
2. 视使用频率补充 React 相关规则或技能。

**价值**：避免 TS 项目必须依赖 Vue 规则，提升通用前端适配性。

### P2：轻量自动化（第三阶段）

1. 增加最小 hooks 能力（优先轻量、可选启用）：
   - `PreToolUse`：危险命令提醒/约束
   - `Stop`：结束前检查提示（如测试、格式化、敏感信息）
2. 后续再逐步引入脚本化校验与测试用例。

**价值**：在不显著增加复杂度的前提下，提高执行稳定性。

### P3：跨 CLI 迁移效率（可选）

1. 新增 `adapters/` 或迁移模板，先打通最小转换链路：
   - Claude Code → Codex CLI
   - Claude Code → Cursor
2. 输出标准化迁移说明，减少手工拷贝与格式调整成本。

**价值**：强化“兼容其他 CLI”这一项目定位。

## 四、建议落地顺序

1. **P0 一致性修复**（文档与能力对齐）
2. **P1 TypeScript 独立规则**（补齐非 Vue 场景）
3. **P2 轻量 hooks**（最小自动化）
4. **P3 迁移适配器**（按需求推进）

## 五、完成标准（验收建议）

- 文档提到的代理/命令在仓库中均可找到对应实现。
- 至少一个非 Vue 的 TypeScript 示例项目可直接套用规则。
- hooks 为可选启用，默认不破坏现有轻量体验。
- 迁移文档含最小可运行示例（而非仅概念说明）。
